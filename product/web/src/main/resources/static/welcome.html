What is Codekvast?

Codekvast identifies Truly Dead Code in your code base, code that you safely can delete, since no-on e is using it.

How?

  • It records when your public Java methods are invoked.
  • It examines your application to find all methods that could be invoked.
  • It uploads the collected data to a server.
  • It visualizes the collected data in a web interface.

Why would I need Codekvast?

  • It Reduces Maintenance Costs. Less code means, well, less maintenance...
  • It Increases Agility. Less code that slows down development.
  • It Improves Quality. Less code means fewer bugs.
  • It Improves Security. Less code means fewer vulnerabilities.

How does it work?

  • Codekvast Collector captures invocations of your business methods with [_extremely_ low overhead].
  • Codekvast Agent combines the collected data with an inventory of your installed application code.
  • Codekvast Agent periodically uploads data for the method invocations as well as the complete set of existing methods to a cloud-hosted Codekvast Server (a data warehouse).
  • Codekvast Server has a real-time web interface for visualizing the captured data, so that you can learn what is your application's Truly Dead Code.
